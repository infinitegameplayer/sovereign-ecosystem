## IV. Architectural Structure (Format 3)

All eligible Sovereign Ecosystem objects may contain three layers.

### Layer 1: YAML Frontmatter (Structured Contrast Fields)

Used primarily for foundational objects.

Recommended fields:
- contrast_not: Commonly confused adjacent concepts.
- false_twins: Look-alike frameworks with structural differences.
- anti_patterns: Degrading behaviors or misapplications.
- boundary_conditions: Operational constraints and edges.
- clarity_triggers: Signals indicating {{AI_INTERFACE_NAME}} should ask follow-up questions.

These fields are optional except for Tier 1 objects.

### Layer 2: Primary Body (Front-Facing Core)

Contains:
- Affirmative definitions
- Operational logic
- Protocols
- Implementation detail
- Constructive framing

Negation is minimal and used only when clarity requires it.

### Layer 3: Footer Block ({{AI_INTERFACE_NAME}} Internal Contrast Layer)

Placed at bottom of file inside a standardized HTML comment wrapper.

Purpose:
- Rich internal reasoning notes
- Edge-case modeling
- Confusion history
- Evolution notes
- Contrast analysis not needed in the front-facing body

This section supports {{AI_INTERFACE_NAME}} cognition and system memory integrity.

## VII. Output Filter Rule

Before responding, {{AI_INTERFACE_NAME}} checks:
- Is negation present?

- Is it repeated?
- Does it anchor imagery in anti-goals?
