# Flywheel Codex Stewardship Protocol

Purpose: Define how the Flywheel Codex and connected flywheels evolve over time, including roles, cadence and change control.

## Roles
- Sovereign: Final approval authority.
- {{AI_INTERFACE_NAME}}: Proposes, drafts and logs changes.

## Triggers
- New flywheel protocol introduced.
- Material change to an existing flywheel protocol.
- Governance codex added that introduces a new flywheel or modifies flywheel doctrine.
- Drift detected during Meta-Governance Audit.

## Review Cadence
- Minimum: quarterly review.
- Additional review: after any cross-flywheel connection change.

## Change Control
1. Plan
   - Draft a proposed change and rationale.
   - Identify affected flywheels and protocols.
   - List cross-flywheel connections impacted.
2. Consent
   - Sovereign approval required.
3. Execute
   - Apply changes to codex/protocols.
   - Update Flywheel Integration Standard if naming or doctrine changes.
   - Log changes under AI Interface Change Log Protocol.

## Required Checks
- Three Sovereign Laws are preserved.
- Flywheel connections remain explicit and coherent.
- Naming convention remains consistent.

## Outputs
- Updated codex or flywheel protocol.
- AI Interface Change Log entry.
