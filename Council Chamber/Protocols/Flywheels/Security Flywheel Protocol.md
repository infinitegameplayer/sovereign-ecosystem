# Security Flywheel Protocol

Purpose: Run layered security checks on-demand and during Meta-Governance Audit.

## Triggers
- Manual Security Check Skill (monthly-ish or when needed).
- Meta-Governance Audit cycle.

## Flywheel Loop
1. Govern: confirm scope + consent boundaries.
2. Identify: assets, dependencies, sensitive zones.
3. Protect: access, integrity and boundary check.
4. Detect: anomalies and drift.
5. Respond: propose fixes (approval-gated).
6. Recover: verify rollback readiness.

## Multi-Agent Phases (Reprogram 3-Hour Base)
Phase 1 - Surface Exploration and Threat Detection
- Multiple agents scan in parallel.
- Reconvene into a single synthesis before moving to Phase 2.

Phase 2 - Deeper Systematic Reprogramming
- Multiple agents scan in parallel.
- Reconvene into a single synthesis before moving to Phase 3.

Phase 3 - Integration, Protection and Future-Proofing
- Multiple agents scan in parallel.
- Reconvene into a single synthesis before action proposals.

## Cross-Flywheel Connections
- North Star: risk alignment with trajectory.
- Conduction Flywheel: Trifecta integrity checks.
- Universal Vault Flywheel: sensitive handling guidance.

## Approval Gate
All actions approval-gated. Outputs are proposals by default.
