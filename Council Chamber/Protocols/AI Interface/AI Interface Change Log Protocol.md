# AI Interface Change Log Protocol

Purpose: Define when AI Interface Change Log entries are required to maintain governance traceability.

## Log When
- Any governance document changes (Constitution, Charter, Codices, Protocols, Skills, Meta-Governance Audit).
- Any system-wide process changes.
- Any North Star structural changes.

## Log Format
- Date
- Trigger
- Action or Decision
- Notes (optional)

Note: Use the Session Closeout Protocol after foundational sessions.

## Rollover Rule
- `Council Chamber/AI Interface/AI Interface Change Log.md` is the active log only.
- Keep a recent operational window in the active log; older entries roll into `Council Chamber/AI Interface/Archive/`.
- Recommended threshold:
  - When the active log exceeds roughly 25 to 40 entries, archive the oldest entries into a dated volume.
- Recommended active window after rollover:
  - Keep the most recent 10 to 15 entries in the active log so it stays easy to scan.
- Archive naming:
  - Use dated volumes such as `AI Interface Change Log - YYYY-MM Volume N.md`.
- Archive location:
  - Keep AI Interface change-log archives in `Council Chamber/AI Interface/Archive/`, not in `Vault`, because they remain part of active governance memory.

## Contrast Layer Integration (Mandatory)
Tier: 1 (foundational).
- Include contrast fields in frontmatter for foundational updates.
- Store deeper contrast reasoning in the footer block when needed.
- Primary body remains affirmative.
