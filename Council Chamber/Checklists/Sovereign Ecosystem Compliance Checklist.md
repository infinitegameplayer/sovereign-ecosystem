# Sovereign Ecosystem Compliance Checklist

Purpose: Ensure new governance and operational artifacts align with core rules.

## Status and Planning Mode
- `status: proposed` and `status: draft` are planning-only
- `status: completed` or equivalent completion state is required before archival scan when the object type uses it
- archived items are excluded from operational scans

## Contrast Layer Integration
- tier is declared for foundational objects when required
- contrast fields are present when required
- primary body remains affirmative

## Type and Domain Clarity
- object type is explicit when needed
- domain or archival path is clear when relevant

## Approval Gates
- no execution without explicit approval where required
- archival and reconciliation actions follow approval-gated rules
