# Update: v1.3.0 — Agent SDK Integration

**Published:** 2026-03-11
**Type:** Core + Skills
**Breaking changes:** None

---

## What Changed and Why

The Claude Agent SDK enables parallel subagent execution, session resumption and session forking as operational capabilities — not experimental features. This update formalizes those capabilities across the Sovereign Ecosystem skill layer.

Before this update, several skills described parallel work in hedged language ("if multiple agents are available") or handled it implicitly. This update makes the parallel architecture explicit and consistent across all skills that benefit from it. It also documents the SDK capabilities in the adapter so the AI interface can reference them without needing to infer from skill prose.

A new Weekly Story skill is added. It was previously out of scope for the Foundation. It now ships as a standard operational skill alongside End-of-Week.

## How to Apply to an Existing Install

If you set up the Sovereign Ecosystem before this update, apply changes in order:

**1. Update `.claude/CLAUDE.md`**

Append the Agent SDK Capabilities section to the end of your `.claude/CLAUDE.md`. Copy from this repo's `.claude/CLAUDE.md` — the section begins with `## Agent SDK Capabilities (Active)`.

**2. Replace four skill files**

Copy the updated SKILL.md files into your local install:
- `Council Chamber/Skills/AI Interface Activation/SKILL.md`
- `Council Chamber/Skills/End-of-Week/SKILL.md`
- `Council Chamber/Skills/Security Check/SKILL.md`
- `Council Chamber/Skills/Session Closeout/SKILL.md`

**3. Add the Weekly Story skill**

Copy the full `Council Chamber/Skills/Weekly Story/` folder into your local install.

Create the output directory if it does not exist:
```
Scriptorium/Weekly Stories/
```

**4. Update Skills Index**

Add to `Council Chamber/Skills/Skills Index.md`:
```
- Weekly Story (active, tier: operational)
```

**5. Update symlinks (if using slash commands)**

If you have `.claude/skills/` symlinks set up, add one for Weekly Story:

Windows (run as Administrator):
```
mklink /J ".claude\skills\Weekly Story" "Council Chamber\Skills\Weekly Story"
```

Mac / Linux:
```
ln -s "Council Chamber/Skills/Weekly Story" ".claude/skills/Weekly Story"
```
