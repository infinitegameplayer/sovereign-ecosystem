# v1.2.0 — Activation Protocol Update

**Date:** 2026-03-11
**Type:** core
**Optional:** no

---

## Summary

The AI Interface Activation Protocol and Skill now use the AI Interface Change Log as the primary callback and orientation source at session start, replacing the Continuity Log as the default carry-forward. A gap-adaptive tier system scales read depth based on how long it has been since the last session — preventing context overload in rapid back-to-back sessions and ensuring full reorientation after long breaks.

The Continuity Log is demoted to a Tier 3-4 source: useful for process patterns after long gaps, not for session-to-session callbacks.
